# 화면 설계서

> 코드넛 채용 지원자 분석 대시보드 (v2.0)

## 1. 화면 정보 구조 (IA)

| 영역 ID | 영역명 | 위치 | 주요 기능 |
|---|---|---|---|
| A | Header | 최상단 | 서비스명, 데이터 기간, 동기화 상태 |
| B | KPI Summary | 헤더 아래 | 3개 핵심 지표 카드 |
| **I** | 공고 사이드 패널 | 좌측 (탭 1) | 공고 리스트 + 선택 시 상세 |
| C | Filter Panel | 좌측 (탭 2) | 그래프 분류·기간 + 직무·사이트 |
| D | Trend Chart | 우측 메인 | 일자별 누적 지원자 꺾은선 |
| E | Job Breakdown | 하단 좌 | 직무별 누적 지원자 가로 막대 |
| F | Site Distribution | 하단 우 | 사이트별 비중 도넛 |
| H | Stacked Bar | 최하단 | 공고별 최종 지원자 (게시일 X) |

## 3. 영역별 상세 설계

### 3.1 A. Header

| ID | 요소 | 설명 |
|---|---|---|
| A-01 | 서비스 제목 | "코드넛 채용 지원자 대시보드" |
| A-02 | 데이터 기간 | `MIN(측정일) ~ MAX(측정일)` 표시 |
| A-03 | 동기화 배지 | 최근 sync 상대시간. 클릭 시 데이터 새로고침 |

### 3.2 B. KPI Summary

| ID | 요소 | 산출 |
|---|---|---|
| B-01 | 코드넛 누적 지원자 | 모든 공고의 MAX(지원자수) 합계 |
| B-02 | 활성 공고 수 | 상태 = '운영중' 공고 수 |
| B-03 | 진행 중인 직무 | 활성 공고의 직무 컬러 칩 표시 |

### 3.3 I. 공고 사이드 패널 (탭 1) — 신규

| ID | 요소 | 설명 |
|---|---|---|
| I-01 | 탭 전환 | "공고" / "필터" 탭. 기본 "공고" |
| I-02 | 공고 검색창 | 공고명 실시간 필터 |
| I-03 | 상태 필터 칩 | 진행중 N / 종료 N 토글 |
| I-04 | 공고 카드 | 상태, 직무(사이트), D+x/30, 누적, 경쟁률 |
| I-05 | 정렬 옵션 | 게시일/지원자/경쟁률 순 (옵션) |
| I-06 | 공고 상세 카드 | 우측 상단. 선택된 공고의 메트릭 표시 |
| I-07 | 진행 프로그레스 바 | 게시일 ~ 마감일 진행률 (마감일 입력 시) |

**공고 카드 표시 정보**:
- 상태 칩 (진행중/종료)
- 직무(사이트) — 예: "프론트(사람인)"
- D+x/30 — 게시일 기준 경과일
- 누적 지원자 수
- 경쟁률 (모집인원 입력 시)

### 3.4 C. Filter Panel (탭 2)

| ID | 요소 | 설명 |
|---|---|---|
| C-01 | 그래프 분류 드롭다운 | 직무별 / 사이트별 / 전체 합계 |
| C-02 | 기간 드롭다운 | 최근 30일(기본) / 전체 / 최근 7일 / 직접 입력 |
| C-03 | 직무 선택 | 그래프 분류에 따라 라디오 또는 체크박스 |
| C-04 | 사이트 선택 | 그래프 분류에 따라 라디오 또는 체크박스 |
| C-05 | 필터 초기화 | 기본값 리셋 (옵션) |

### 3.5 D. Trend Chart (메인 꺾은선)

| ID | 요소 | 설명 |
|---|---|---|
| D-01 | X축 | 측정일. 30일 윈도우 (기본) |
| D-02 | Y축 | 지원자 수 (누적). 자동 스케일 |
| D-03 | 데이터 시리즈 | 분류 기준에 따라 라인 수 결정 |
| D-04 | 공고 선택 모드 | I-04에서 공고 선택 시 해당 공고 단일 라인 |
| D-05 | 툴팁 | Hover 시 측정일, 분류명, 누적값 |
| D-06 | 범례 | 차트 하단. 클릭 시 시리즈 토글 |
| D-07 | Empty State | "선택된 조건의 데이터가 없습니다" |

### 3.6 E. 직무별 막대

- 가로 막대, 지원자수 **내림차순**
- 직무 컬러 적용
- 막대 클릭 시 해당 직무로 메인 그래프 필터링 (C-03 동기화)

### 3.7 F. 사이트별 도넛

- 사이트 컬러 적용
- 범례에 사이트명 + 퍼센트
- 조각 클릭 시 해당 사이트로 필터링 (C-04 동기화)

### 3.8 H. 공고별 Stacked Bar

| ID | 요소 | 설명 |
|---|---|---|
| H-01 | 직무 드롭다운 | 단일 직무 선택 |
| H-02 | X축 | 공고 게시일 (오름차순). 같은 게시일은 하나 막대 |
| H-03 | Y축 | 공고 최종 누적 지원자 수 |
| H-04 | Stack 구성 | 사이트별 색상 (사람인→잡코리아→로켓펀치→그룹바이) |
| H-05 | Stack 라벨 | 각 stack 내부에 수치 |
| H-06 | 총합 라벨 | 막대 상단에 게시일별 총합 |
| H-07 | X축 보조 | "공고 N건" 표시 |

## 4. 필터 동적 인터랙션

「그래프 분류」 드롭다운 선택값에 따라 직무·사이트 UI가 변환됩니다.

| 그래프 분류 | 직무 UI | 사이트 UI | 메인 그래프 |
|---|---|---|---|
| 직무별 | 라디오 (단일) | 체크박스 (다중) | 직무 1개 × 사이트 N개 = N개 라인 (사이트 컬러) |
| 사이트별 | 체크박스 (다중) | 라디오 (단일) | 사이트 1개 × 직무 N개 = N개 라인 (직무 컬러) |
| 전체 합계 | 체크박스 (다중) | 체크박스 (다중) | 합산 1개 라인 |

### 사용 예시

- **프론트의 사이트별 비교**: 그래프 분류 "직무별" + 프론트 선택 + 사람인·잡코리아 체크
- **사람인의 직무별 비교**: 그래프 분류 "사이트별" + 사람인 선택 + 전체 직무 체크

## 5. 인터랙션 시나리오

| No. | 시나리오 | 동작 흐름 |
|---|---|---|
| S1 | 공고 상세 분석 | I 영역 공고 카드 클릭 → 우측 상단 상세 카드 표시 → 메인 그래프 D가 해당 공고 단일 라인으로 전환 |
| S2 | 필터 기반 비교 | "필터" 탭 → 그래프 분류 + 직무 + 사이트 선택 → D 라인 재구성 |
| S3 | 게시일별 효율 비교 | H 영역 직무 드롭다운 선택 → Stacked Bar 렌더링 |
| S4 | E 막대 빠른 필터링 | E 막대 클릭 → D와 C-03이 자동 동기화 |
| S5 | 공고 검색 | I-02 검색창에 키워드 입력 → 리스트 실시간 필터 |
| S6 | 데이터 새로고침 | A-03 동기화 배지 클릭 → API 재호출 → 위젯 재렌더링 |

## 6. 데이터 가공 로직 요약

| 영역 | 산출 |
|---|---|
| B-01 | `Σ(공고별 MAX(지원자수))` 전체 |
| B-02 | `COUNT(jobs WHERE 상태='운영중')` |
| B-03 | `DISTINCT(activeJobs.직무구분)` |
| D | `[측정일 × 분류기준] GROUP BY → SUM(지원자수)`. 30일 윈도우 |
| E | `직무별 SUM(공고별 MAX)`. 내림차순 정렬 |
| F | `사이트별 SUM ÷ 전체 × 100` |
| H | `[게시일 × 사이트] GROUP BY → SUM(공고별 MAX)` |

## 7. 예외 처리

- 모집인원 부재 → 경쟁률 "-"
- 마감일 부재 → 진행률 "-", D+x만 표시
- 지원자수 = 0 → 정상 데이터, 단 막대/도넛에서 비표시
- 측정 1회 공고 → 메인 그래프에 점만 표시
- 필터 결과 0건 → Empty State
- 동기화 실패 → 헤더 경고 배지
