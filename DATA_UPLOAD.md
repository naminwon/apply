# 데이터 입력 매뉴얼 — 코드넛 채용 지원자 대시보드

이 문서는 **Google 스프레드시트(`채용_지원자_데이터_미러`)에 행을 직접 추가**해 대시보드 데이터를 업데이트하는 담당자를 위한 가이드입니다.

> **한 줄 요약**: 시트에 행만 정확히 추가하면 끝. 대시보드는 **시트가 진실의 원천**이며, 별도의 업로드·빌드 작업이 필요 없습니다. 입력 후 약 5분 이내(브라우저 새로고침 시 즉시) 반영됩니다.

---

## 1. 시트 접근

- 스프레드시트 이름: **`채용_지원자_데이터_미러`** (Google Drive)
- 권한: 편집자 권한이 있어야 입력 가능. 권한 요청은 [관리자 mwna@codnut.com] 에게.
- 시트 탭이 세 개 있습니다.
  - `jobs_master` — 공고 마스터 (공고 1건당 1행)
  - `applicants_log` — 측정 로그 (측정 1건당 1행)
  - `sync_log` — 동기화 이력 (수동 입력 X, 자동 기록)

---

## 2. 신규 공고 등록 — `jobs_master` 시트

새 채용 공고가 사이트(사람인/잡코리아/로켓펀치/그룹바이)에 게시되면 이 시트에 한 행을 추가합니다.

### 2.1 컬럼 입력 규칙

| 컬럼 | 필수 | 형식 / 허용값 | 예시 |
|---|---|---|---|
| `게시일` | ✅ | `yyyy-MM-dd` 또는 날짜 셀 | `2026-05-19` |
| `공고명` | ✅ | 자유 문자열 (URL은 넣지 않음 — 깔끔한 제목만) | `[신입/경력] 코드넛과 함께할 UI/UX 디자이너` |
| `사이트` | ✅ | **정확히** `사람인` / `잡코리아` / `로켓펀치` / `그룹바이` 중 하나 | `사람인` |
| `직무구분` | ✅ | **정확히** `AI` / `백엔드` / `프론트` / `기획자` / `디자이너` 중 하나 | `디자이너` |
| `공고URL` | ✅ | 사이트별 채용 공고 URL 전체 | `https://www.saramin.co.kr/zf_user/jobs/view?rec_idx=...` |
| `상태` | ✅ | `운영중` 또는 `마감` | `운영중` |
| `마감일` | ❌ | `yyyy-MM-dd`. 미정이면 빈 셀 | `2026-06-30` |
| `모집인원` | ❌ | 정수 | `2` |
| `광고집행` | ❌ | 정수(원). 미집행이면 빈 셀 | `1000000` |
| `비고` | ❌ | 자유 메모 | `리뉴얼 공고` |

### 2.2 절차

1. `jobs_master` 시트 마지막 행 아래에 빈 행을 만든다.
2. 위 표대로 한 행 채운다.
3. 저장(자동 저장됨). 끝.

### 2.3 자주 하는 실수

- ❌ `백엔드`를 `벡엔드`로 오타 → 코드가 한 가지 오타(`벡엔드→백엔드`)만 자동 보정합니다. 그 외 오타(`벡엔트`, `Backend` 등)는 보정되지 않습니다.
- ❌ 사이트명에 공백/괄호 포함 (`사람인 `, `사람인(saramin)`) → 정확히 일치하지 않으면 컬러가 회색 폴백으로 표시됩니다.
- ❌ `공고URL`을 빈 칸으로 두기 → `applicants_log`와 JOIN이 안 되어 일부 지표(상태·광고집행)가 비게 됩니다.
- ❌ `상태` 값에 `진행중`이나 `종료`처럼 다른 단어를 쓰기 → `운영중`/`마감` 두 값만 허용.

## 3. 지원자수 측정 입력 — `applicants_log` 시트

**주 2회 정기 측정**(보통 월요일/목요일). 측정 회차마다 각 공고별로 한 행씩 추가합니다.

### 3.1 컬럼 입력 규칙

| 컬럼 | 필수 | 형식 | 예시 |
|---|---|---|---|
| `log_id` | ✅ | `{측정일yyyyMMdd}_{공고제목}` 형식. 중복 금지 | `20260519_[신입/경력] 코드넛과 함께할 UI/UX 디자이너` |
| `지원자수` | ✅ | 정수(누적값). 사이트 관리자 페이지에서 확인 | `153` |
| `게시일` | ✅ | `jobs_master`의 동일 공고 `게시일`과 일치 | `2026-01-02` |
| `측정일` | ✅ | 오늘 날짜 `yyyy-MM-dd` | `2026-05-19` |
| `공고명` | ✅ | **공고제목 + 한 칸 띄우고 (URL)** 형태 — 아래 3.3 참조 | `[신입/경력] ... 디자이너 (https://www.saramin.co.kr/...)` |
| `사이트` | ✅ | `jobs_master`와 동일 enum | `사람인` |
| `직무구분` | ✅ | `jobs_master`와 동일 enum | `디자이너` |
| `비고` | ❌ | 자유 메모 | `광고 시작` |

### 3.2 측정 절차 (한 회차)

1. 각 사이트의 채용 관리자 페이지에서 **운영중인 공고**의 현재 지원자수를 확인.
2. `applicants_log` 시트 마지막 행 아래로 공고 수만큼 행 추가.
3. 공고 1건마다 한 행씩 위 표에 따라 입력.
4. **이전 측정의 행과 비교**해 누적값이 줄어들지 않았는지 확인 (지원자수는 단조증가).
5. 저장 완료.

### 3.3 ⚠️ 가장 중요한 규칙 — `공고명` 컬럼 형식

대시보드는 `applicants_log.공고명` 안에서 **`(URL)`** 패턴으로 URL을 추출해 `jobs_master`와 매칭합니다. 이게 깨지면 데이터가 "공고 없음" 상태로 분리됩니다.

올바른 형식:
```
[신입/경력] 코드넛과 함께할 UI/UX 디자이너 (https://www.saramin.co.kr/zf_user/jobs/view?rec_idx=52709684&view_type=etc&ref=recruit_manage)
```

규칙:
- 제목 뒤에 **공백 한 칸 + `(`** + URL + `)` 로 묶기.
- URL은 `https://` 또는 `http://` 로 시작해야 하고, `)` 가 닫혀 있어야 합니다.
- 동일 공고는 매 측정마다 **같은 URL**을 사용. 다른 사이트가 같은 채용을 광고하더라도 URL이 다르면 별개 공고로 집계됩니다.

가장 안전한 방법: `jobs_master`에 등록된 동일 공고의 `공고URL` 셀을 복사 → `공고명` 끝에 ` (URL)` 형식으로 붙여 넣기.

### 3.4 `log_id` 작성 팁

- 형식은 자유지만 **유일성**이 중요합니다. 같은 `log_id`가 두 번 들어가면 어느 쪽이 채택될지 보장되지 않습니다.
- 권장: `{측정일yyyyMMdd}_{공고제목 앞부분}`
- 같은 회차에 같은 공고가 두 번 측정될 일은 없으므로 위 형식으로 충돌 가능성 거의 없음.

### 3.5 자주 하는 실수

- ❌ `공고명`에 URL을 빠뜨림 → 대시보드에서 "공고 미연결" 상태가 되어 사이드 패널 공고 목록에 안 나타날 수 있음.
- ❌ `측정일`을 시트 입력일이 아닌 "이전 회차 날짜"로 잘못 입력 → 그래프의 측정일 축이 어긋남.
- ❌ 마감된 공고를 계속 측정해서 행 추가 → 누적값이 변하지 않으므로 측정 의미 없음. **`jobs_master.상태`를 `마감`으로 바꾼 뒤로는 더 입력하지 않음**.
- ❌ `지원자수`를 "이번 회차에 새로 들어온 수"로 입력 → **누적값**이어야 합니다. 그래프 로직이 누적 단조증가를 가정합니다.
- ❌ `log_id` 중복 입력 (예: 같은 측정 회차의 같은 공고를 두 번 기록).

---

## 4. 공고 마감 처리

채용이 끝났을 때:

1. `jobs_master`에서 해당 공고 행의 `상태` 셀을 `운영중` → `마감`으로 변경.
2. (선택) `마감일` 셀에 실제 마감 날짜 입력.
3. `applicants_log`에는 더 이상 새 행을 추가하지 않음.

→ 대시보드의 **활성 공고 수(B-02)** 가 자동으로 감소합니다.

## 5. 입력 후 대시보드 반영 시점

| 경로 | 반영 시간 |
|---|---|
| 브라우저 강제 새로고침 (Cmd+Shift+R) | 즉시 |
| 일반 새로고침 (F5) | 즉시~5분 (React Query 캐시 만료 시점에 따라) |
| 그냥 두기 | 최대 5분 |

> **재배포 불필요**: 시트 → Apps Script → 브라우저 fetch 경로라서, GitHub에 코드를 푸시하거나 빌드를 다시 돌릴 필요가 **없습니다**. 시트만 정확하면 됩니다.

---

## 6. 입력 직전 체크리스트

신규 공고 등록 시:
- [ ] `사이트`·`직무구분` 값이 정확한 enum인지
- [ ] `공고URL`을 채웠는지
- [ ] `상태`가 `운영중`인지
- [ ] `게시일`이 `yyyy-MM-dd`로 잘 들어갔는지

정기 측정 입력 시:
- [ ] 모든 운영중 공고에 대해 한 행씩 추가했는지
- [ ] `공고명` 셀에 ` (URL)` 형식의 URL이 붙어 있는지
- [ ] `지원자수`가 이전 회차보다 같거나 큰지 (단조증가)
- [ ] `측정일`이 오늘 날짜인지
- [ ] `log_id`가 유일한지

## 7. 문제가 생겼을 때

| 증상 | 우선 확인 |
|---|---|
| 새 공고가 대시보드에 안 보임 | 5분 대기 후 강제 새로고침. 그래도 안 나오면 `사이트`·`직무구분`·`상태` 값 오타 확인 |
| 새 측정값이 그래프에 안 보임 | `공고명` 안의 URL 형식(`(https://...)`) 점검. URL이 `jobs_master.공고URL`과 정확히 일치하는지 확인 |
| 그래프 색이 회색으로 폴백됨 | `사이트` 또는 `직무구분`이 허용 enum과 정확히 일치하지 않음 |
| 대시보드 자체가 안 열림 | 인수인계 문서(`HANDOVER.md`)의 트러블슈팅 섹션 참조 |
